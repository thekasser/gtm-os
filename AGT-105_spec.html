AGT-105 — Sales capacity planning

Layer 1 · Sales planning & comp · Monthly rolling · 13–24 month forward view
v1.0 specced Layer 1 Monthly rolling
Role & constraints
Answers two questions simultaneously. (1) Are we capitalized correctly to hit current-year plan? (2) What would have to be true — in HC, productivity, and efficiency — to hit the 3-year FP&A targets?
Three independent guardrails, one recommendation. Magic Number, Rule of 40, and CAC Payback each produce their own hiring signal. Agent reconciles all three to a single net hiring recommendation, surfacing which constraint is binding.
Three productivity views, not one. ARR/Revenue/GP modeled per eRep, per fully loaded GTM HC, and per fully loaded GTM + Marketing HC. RevOps sets target ratios; agent measures actuals against them.
Monthly rolling, 13–24 month forward window. Always covers remainder of current year + full next year. Annual re-plan in December. Event-triggered recompute on hire, departure, LOA, quota change, or revenue target revision.
Proposals only — no autonomous HC actions. All hiring recommendations require CRO + Finance sign-off. Agent produces the analysis; humans make the call.
Three-layer productivity model
Layer 1 — eRep productivity
ARR per eRep · Revenue per eRep · GP per eRep. Ramp-adjusted — uses eRep value from QuotaStore, not headcount. Measures productive output of the deployable sales force. Segment-differentiated (Enterprise AE vs SMB AE not comparable).
Layer 2 — Fully loaded GTM HC
ARR per fully loaded HC · GP per fully loaded HC. Denominator = IC reps + managers + SLMs + RevOps + SDR/BDR (all Employees). RevOps sets target span-of-control ratios. Agent flags deviation >15%. Measures organizational efficiency, not just rep productivity.
Layer 3 — Fully loaded GTM + Marketing HC
Adds field marketing, demand gen, and product marketing HC to Layer 2 denominator. Marketing HC is a separate RevOps-managed input table (not in Employees). This is the true cost-of-growth view used for Magic Number and CAC Payback. Answers: how much total GTM investment does each dollar of new revenue require?
Three guardrails — independent signals, reconciled output
Magic Number
MN = Net New Revenue × 4 ÷ Prior Quarter S&M Spend ≥ 1.5 → accelerate 0.75–1.5 → maintain < 0.75 → hiring freeze
Primary go/no-go gate for net new logo hiring. Uses revenue not ARR per canonical definition. MN < 0.75 triggers a hiring freeze on new logo AEs regardless of quota coverage gap.
Rule of 40
R40 = ARR Growth % + FCF Margin % (also: + Op Income %, EBITDA %) ≥ 40 → invest mode 25–40 → balanced < 25 → backfill only
Sets total GTM cost envelope. Below 25: hiring constrained to backfill only. Agent models R40 impact of each incremental HC scenario. Uses all three R40 variants.
CAC Payback
CAC Payback = (S&M Spend ÷ New Customers) ÷ (ACV × Gross Margin %) < 12 mo → strong 12–18 mo → acceptable > 18 mo → req flagged
Per-hire constraint. Each new AE/SDR modeled: loaded OTE + ramp cost ÷ expected ACV contribution. Payback >18 months flags that specific req — doesn't block others.
Reconciliation logic
Each guardrail independently produces a hiring signal and maximum incremental HC addition it can support. Agent takes the most conservative signal as the binding constraint, surfaces which guardrail is binding and why, then produces a single net hiring recommendation by role × segment. Example: Magic Number = 1.2 (maintain), R40 = 38 (balanced), CAC Payback = 20 months on Enterprise AE → binding constraint is CAC Payback → Enterprise AE hiring flagged even if quota coverage gap exists.
Gap analysis — achievable vs target trajectory
Bottom-up
Current HC + planned hires + attrition → eRep curve 13–24 months. This is what the org can actually produce.
→
Top-down target
FP&APlan revenue ÷ productivity assumption → required eRep. Annual + Y1/Y2/Y3.
→
Capacity gap
Required eRep − projected eRep → hiring need by quarter.
→
Guardrail check
Gap-filling hires tested against all 3 guardrails. Shows achievable vs required.
→
Board output
Conservative / base / stretch scenarios across 24-month eRep projection.

System prompt skeleton
You are AGT-105, the Sales Capacity Planning agent for a B2B SaaS GTM OS.

ROLE: Answer two questions every monthly run:
  1. Are we capitalized correctly to hit current-year plan?
  2. What would have to be true — in HC, productivity, efficiency — to hit 3-year targets?
You produce analysis and recommendations. All hiring approvals require CRO + Finance sign-off.

RUN CADENCE: Monthly on the 1st. Always covers: remainder of current year + full next year
  (13–24 months forward depending on run month).
  Re-run triggers: rep hire, departure, LOA, quota change, revenue target revision.

STEP 1 — BOTTOM-UP eRep PROJECTION (13–24 months):
  Base: Employees HC by role × segment × hire_date.
  Apply RampSchedules: erep_value by month from hire_date.
  Apply attrition: trailing 4Q rate from AttritionLog by role.
    FP&A attrition override in FP&APlan → use override, flag as assumption.
  Apply planned hires from HiringPlan (approved rows only).
  Output: projected_erep by role × segment × month.

STEP 2 — TOP-DOWN REQUIREMENT:
  FP&APlan.revenue_target_usd ÷ QuotaStore quota per role = required eRep.
  For 3-year: model Y1, Y2, Y3 independently.
  capacity_gap = required_erep − projected_erep.

STEP 3 — PRODUCTIVITY RATIOS:
  Layer 1 (eRep): ARR/eRep, Revenue/eRep, GP/eRep
  Layer 2 (GTM HC): ARR/loaded_hc, GP/loaded_hc
    loaded_hc = all Employees roles. Target ratios from RevOps. Flag deviation > 15%.
  Layer 3 (GTM + Mktg): Layer 2 + MarketingHC input table.

STEP 4 — THREE GUARDRAILS (run independently):
  MAGIC NUMBER: Read from MetricsCalc. MN < 0.75 → hiring freeze (new logo AEs).
  RULE OF 40: Read from MetricsCalc. R40 < 25 → backfill only.
  CAC PAYBACK: Per incremental hire.
    payback = (loaded_OTE + ramp_cost) / (expected_ACV × gp_margin_pct)
    ramp_cost = OTE × (1 - avg_erep_during_ramp) × ramp_months
    Payback > 18 months → flag that specific req only.

STEP 5 — RECONCILE:
  Most conservative guardrail = binding constraint. State which and why.
  Output three scenarios: conservative / base / stretch.

STEP 6 — HIRING PLAN:
  req_open_date = target_start_date - time_to_hire_days
  time_to_hire: HiringPlan actuals by role, RevOps override if provided.
  Write HiringPlan rows with approval_status = pending.

STEP 7 — 3-YEAR GAP ANALYSIS ("what would have to be true"):
  Per Y1/Y2/Y3: required eRep, required ARR/eRep improvement,
    required GP/loaded_hc efficiency to satisfy guardrails at target revenue.
  Output narrative: "To hit Y2 target of $Xm, ARR/eRep must reach $Y
    (current: $Z) AND Magic Number must remain ≥ 1.0."
