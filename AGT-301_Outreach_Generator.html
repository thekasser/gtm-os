L2 wire items (v22)
Wire 1 — sequence_type written to CadenceEventLog at draft time
When AGT-301 drafts a sequence, it reads Accounts.abm_active and Accounts.abm_tier and writes CadenceEventLog.sequence_type:
 · abm_active = TRUE AND abm_tier = '1:1' → sequence_type = 'abm_bespoke'
 · abm_active = TRUE AND abm_tier = '1:few' → sequence_type = 'abm_cluster'
 · cadence_type = 'nurture' (AGT-304 sequences) → sequence_type = 'nurture'
 · All other → sequence_type = 'standard'
Sequence content and step cadence are unchanged. The field is a classification label for AGT-303 analysis — not a behavior modifier.
Wire 2 — priority_score_at_queue written to CadenceEventLog at queue time
When AGT-301 registers a sequence with AGT-302 (Cadence Coordinator), it reads AccountPriorityScore.composite_score for the account and writes it to CadenceEventLog.priority_score_at_queue. Score is snapshotted at queue time — it does not update as the score changes during the sequence. Enables AGT-303 to analyze whether high-priority accounts received timely outreach relative to their score at queue. If no AccountPriorityScore record exists for the account, field is NULL — no blocking behavior.
Existing L2 ripple (v18, unchanged): When Accounts.abm_active = TRUE, AGT-301 runs a deeper enrichment pull across all 6 enrichment categories + recent news/triggers before drafting each step. ABM sequence template and step cadence are unchanged — personalization depth only. Queue order: highest AccountPriorityScore.composite_score drafted first across all queued sequences.
Purpose

AGT-301 generates personalized multi-step outreach sequences triggered by lead routing, state transitions from AGT-302, or rep on-demand requests. It owns 4 cadence types and produces sequences across 5 channels. Every sequence is registered with AGT-302 (Cadence Coordinator) as the single source of truth for active cadences. AGT-301 never sends — it drafts and queues. AGT-302 owns all send execution.

Sequence state transitions

A reply or meeting doesn't end the relationship with the sequence system — it transitions the prospect into a different cadence. AGT-302 owns transition execution; AGT-301 generates the new sequence on signal.

SignalTransition
Positive sentiment replyPause cold cadence → AGT-302 queues Post-Meeting Follow-up prep if meeting booked, or rep-action prompt if not
Negative / neutral reply (door still open)Cold Outreach → Objection Handling cadence (AGT-301 generates on AGT-302 signal)
Meeting bookedCold Outreach → Pre-Meeting Prep cadence
Meeting completed — no dealPre-Meeting → Post-Meeting Follow-up cadence
Opportunity createdAll outreach sequences terminate → AGT-405 Meeting Opportunity Prep takes over
Prospect unsubscribesHard stop. All sequences terminated. Flag propagates to AGT-302 compliance gate — cannot be re-enrolled.
Rep manually opts outHard stop. Sequence terminated. Logged in CadenceEventLog.
