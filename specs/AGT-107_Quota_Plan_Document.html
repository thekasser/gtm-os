Purpose

AGT-107 generates legally deployable quota plan documents from AGT-101 and AGT-102 approved outputs. It is a document execution agent — not a planning agent. It never sets quota, never designs comp plans, and never modifies plan terms. Its job begins when human approvals are complete and ends when the rep has e-signed the document. Plan is not legally active until e-signature is received. AGT-104 audits signature completeness as part of its plan approval gate control.

E-signature integration (DocuSign or equivalent) is a v1 hook — the field model and state machine are specced now; the API integration is wired in the live deployment. In pre-integration operation, HR manually marks signature_received = TRUE in QuotaPlanDocLog and AGT-104 reads that field.
Mid-year amendment logic

When AGT-104 exception path fires a quota amendment, AGT-107 evaluates the change scope and recommends a format. RevOps makes the final call before the document is generated.

Agent recommendation: Addendum
Recommended when the amendment is quota-only — quota target changes but no structural plan terms change. Kickers, accelerators, draw terms, territory, and payout curve are all unchanged. Addendum is a short delta document: shows prior quota vs new quota, effective date of change, reason code from ExceptionLog, and RevOps approver. Attached to the original signed plan as an amendment exhibit. Original plan PDF is not replaced.
Agent recommendation: Full reissue
Recommended when structural plan terms change — new kicker program, territory reassignment, draw terms modification, payout curve change, or role change. Full reissue voids and archives the original plan, generates a complete new document with updated terms, and requires a new e-signature. Prior plan is marked superseded in QuotaPlanDocLog. Rep and manager notified of reissue requirement.
RevOps human selects addendum or full reissue from the agent recommendation before document generation begins. Agent never auto-selects. Selection is logged in QuotaPlanDocLog with the approver's ID and rationale. If RevOps overrides the recommendation, the override reason is logged.
Recommendation is deterministic, not probabilistic. The criteria above are hard rules — if any structural term changed, the agent always recommends full reissue. RevOps may override (e.g. territory change was minor and agreed verbally) but must log the rationale.

AGT-104 integration

AGT-107 creates a new dependency surface for AGT-104's plan approval gate control. Two existing AGT-104 controls now read from QuotaPlanDocLog:

AGT-104 controlWhat it checks
Plan approval gateQuotaPlanDocLog.doc_status = 'active' for all reps in the cohort before AGT-103 payouts are released. Reps with pending_signature are individually held, not the full cohort.
Signature completeness auditQuotaPlanDocLog WHERE doc_status = 'overdue_signature' — flags in CompAuditLog, notifies manager and HR. AGT-104 does not hold payouts beyond the existing payout hold — it audits and escalates.
