AGT-704 — Business Review Orchestrator

Layer 7: Measurement & Feedback · Cadenced synthesis layer for WBR, MBR, and exec QBR
L7 · Agent 04 Specced · v21 Schema adds: BusinessReviewLog
Purpose

AGT-704 is the synthesis and orchestration layer for GTM business reviews. It does not compute metrics, detect patterns, or generate coaching recommendations — those functions belong to AGT-701, AGT-702, and AGT-703. AGT-704 reads from those agents' output tables, validates data currency before pulling, and assembles stakeholder-differentiated review artifacts on a defined cadence. Three output cadences — WBR, MBR, exec QBR — serve different audiences with different detail levels and formats. It owns action item assignment and calibration signal escalation within each review cycle.

Core design principle

AGT-704 never recomputes what an upstream agent owns. If MetricsCalc is stale, it does not derive its own Magic Number. If ChurnRiskLog is stale, it does not approximate NRR risk. It flags staleness, holds the artifact, and alerts RevOps.

Single write owner per table. AGT-704 reads and synthesizes — never recomputes.
Pre-flight staleness check — all cadences

Blocking gate before any artifact generation. Does not begin until check passes or RevOps override is received.

1
Check required sources — verifies each source table has been updated within its expected refresh window.
2
Stale source detected — point alert to owning agent AND RevOps simultaneously. Specifies: which table, last refresh timestamp, expected window, which artifact is blocked.
3
Resolution — (A) Owning agent refreshes: check re-runs, generation proceeds. (B) RevOps override: acknowledges staleness, approves generation. Artifact flagged data_staleness_acknowledged = TRUE.
4
Hard gate. Artifact never auto-generates with unacknowledged stale data.

Artifact model — hybrid

WBR artifacts provide narrative context (what happened week by week). All metric sections pull fresh from source tables. WBR narrative informs the story; canonical tables govern the numbers. MBR never summarizes WBR metric figures.

MBR sections
1
Month narrative summary — synthesized from 4 WBR artifacts. Major wins, losses, inflection points. AGT-704 generates; human edits before distribution.
2
GTM efficiency metrics — fresh pull from MetricsCalc. Magic Number, NRR, GRR vs targets. R40 variants if month-end = quarter-end. Plan vs actual (current FP&APlan version + board plan delta). QoQ trend from MetricsCalc history.
3
Win-loss summary — fresh pull from WinLossLog. Month closed deals by outcome. Top loss reasons by segment and vertical. Persona coverage at close stats. Rep-level win rate vs benchmark — manager-facing only, not surfaced to reps in MBR.
4
Forecast accuracy review — fresh pull from ForecastAccuracyLog. AGT-402 and AGT-404 accuracy for the closed month. Bias detection flags. Model divergence. Which model more accurate YTD.
5
NRR health and expansion pipeline — fresh pull from ChurnRiskLog and ExpansionLog. NRR rolling 3-month trend. GRR trend. Expansion plays opened vs closed vs suppressed. Top at-risk renewals next 90 days.
6
Coaching and performance patterns — fresh pull from RepCoachingLog and RepSkillAssessmentLog. Persistent patterns (3+ weeks unresolved) by role. SLM escalations in the month. Skill gap trends. Aggregate patterns only — no individual rep detail at MBR level.
7
Open calibration signals — fresh pull from CalibrationSignalLog. All open signals with status, age, owning agent, recommended action. Signals open >60 days flagged separately for explicit decision. New signals since last MBR highlighted.
8
Action items — carry-forward from prior MBR with status update + new items from this MBR. Owner confirmed before distribution.

Action item model

AGT-704 assigns suggested owners by routing rules. Humans confirm before artifact distributes. Unresolved items carry forward into the next review cycle — public visibility at cadence is the accountability mechanism.

Action item typeDefault ownerEscalation path
Pipeline coverage gapSLM for affected segmentCRO if gap persists 2+ WBRs
At-risk renewal — no open oppAM for accountSLM if renewal ≤30 days with no opp
CalibrationSignalLog — ICP or forecast biasRevOpsCRO + Finance at MBR if open >60 days
CalibrationSignalLog — quota methodologyRevOps + CROBoard-level if flagged in exec QBR
Capacity / hiring gapSLM + HR via RevOpsCRO if gap affects current quarter coverage
Forecast accuracy — systematic biasRevOpsCRO at MBR if bias persists 3+ periods
Rep coaching escalation (SLM-level)SLMCRO at exec QBR if pattern persists QoQ
Items unacknowledged across 2 consecutive cycles escalate one level in the routing hierarchy automatically. AGT-704 does not chase owners between cycles — carry-forward visibility is the enforcement model.
Cadence discipline

AGT-704 stays on cadence. No unscheduled exec artifacts. Mid-period metric deterioration is handled by AGT-702 breach alerting to RevOps. RevOps triages and determines whether to escalate manually. The OS does not automate that escalation decision.

Design guardrails
Never recomputes what an upstream agent owns. Stale source = artifact holds.
Action item ownership confirmed by humans before distribution. Hard gate.
No individual rep detail in MBR or exec QBR. Aggregate patterns only.
No unscheduled exec artifacts. AGT-704 stays on cadence regardless of signal severity.
