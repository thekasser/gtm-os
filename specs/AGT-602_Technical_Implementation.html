AGT-602 · Layer 6 · Customer Lifecycle
Technical Implementation Agent
Manages the technical workstream of customer onboarding. Active only when implementation services are on the quote. Runs in parallel with AGT-601 from close. Technical owner is the Implementation Engineer (or CSM when no dedicated IE). Reads the MAP and ImplementationSOW as its source of truth. When AGT-602 is not active, AGT-601 covers simplified technical milestones directly.
New agent New tables: ImplementationSOW, TechnicalMilestoneLog Parallel to AGT-601 — MAP is shared coordination layer
Activation gate: AGT-602 activates at deal close when QuoteLog contains an implementation services line item OR when ImplementationSOW record exists for the account. Not active = AGT-601 covers lightweight technical milestones (M1 only) with CSM as owner.

Relationship to AGT-601
Shared MAP coordination
Both agents read MutualActionPlan. AGT-602 reads technical win details, integration requirements, and customer technical stakeholder map. AGT-602 writes technical milestone completion back to MAP (via TechnicalMilestoneLog). AGT-601 reads AGT-602 progress to update M1 milestone status without duplicating the technical workstream.
CSM visibility
AGT-601 surfaces AGT-602 technical status to the CSM in a simplified summary — CSM is the customer-facing PM and needs to know what's happening technically without owning the detail. IE owns AGT-602 operational detail; CSM owns customer communication about it.
