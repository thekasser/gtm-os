AGT-703 — Win-Loss & Forecast Accuracy

Layer 7: Measurement & Feedback · Pattern analysis on deal outcomes and forecast quality · Calibration signal generator
L7 · Agent 03 Built · v31 Owns: WinLossLog, ForecastAccuracyLog, CalibrationSignalLog
v31 — promoted from Specced to Built. AGT-703 produces the calibration signals that propagate back into AGT-101 (quota), AGT-201 (ICP), AGT-402 (forecast), AGT-403 (competitive), AGT-404 (top-down forecast). Without AGT-703 Built, the OS has no closed-loop learning — plays are designed and forecasts produced but bias and pattern signal don't return upstream. AGT-703 is read by both AGT-704 (business review synthesis) and AGT-901 (pipeline brain) via the WinLossLog brain-ready view defined in the v30 contract.
Purpose

AGT-703 owns two distinct but related measurement functions: win-loss pattern analysis across closed deals, and forecast accuracy measurement across both the bottoms-up (AGT-402) and top-down (AGT-404) models. It detects systematic patterns in deal outcomes and forecast deviations, generates calibration signals for upstream agents and L0 config tables, and surfaces those signals through AGT-704 MBR synthesis. AGT-703 flags signals — humans decide whether and how to act on them. It does not write to ICP, quota, or competitive KB tables directly.

Two functional modules
Module A Win-Loss Analysis

Pattern detection across closed-won and closed-lost deals. Derives loss reasons from deal data when rep entry is missing. Identifies systematic gaps in win rate by dimension. Generates calibration signals to ICP model, competitive KB, and sales methodology.

Module B Forecast Accuracy

Measures AGT-402 bottoms-up and AGT-404 top-down forecast accuracy independently, then compares them. Isolates rep commit accuracy as a distinct coaching input to AGT-701. Identifies systematic bias patterns (consistent over vs under-forecast).

Both modules write to separate tables (WinLossLog, ForecastAccuracyLog) but share CalibrationSignalLog as the unified output surface for upstream recommendations.
Module A — Win-Loss Analysis
Input model — how AGT-703 gets loss reason data

Rep entry is optional. The agent always attempts to derive loss reason from deal data first; rep entry enriches and validates but is never a hard dependency.

1
Agent-derived loss reason — primary source. AGT-703 reads stage history (where did deal stall), ConvIntelligence (competitor mentions, unaddressed objections, sentiment trajectory), QuoteLog (approval exceptions, discount depth at loss), AGT-401 deal health dimension scores at close. Derives a structured loss reason with confidence rating.
2
Rep-entered loss reason — enrichment layer. If rep has logged a loss reason in Opportunities (structured field — not a free-text note), AGT-703 compares it against the agent-derived reason. Agreement = higher confidence. Disagreement = flagged for RevOps review — systematic disagreement between rep-stated and agent-derived reasons is itself a signal.
3
Gap handling — if no rep entry and agent confidence is low (insufficient ConvIntelligence data, no stage history detail), the deal is logged in WinLossLog with analysis_confidence = low and excluded from pattern aggregation. Low-confidence records accumulate — if a rep's deals consistently produce low confidence, AGT-701 is notified as a data quality coaching signal.
Win-loss analysis dimensions
Segment
Win rate by SMB / Mid-Market / Enterprise. Cross-tabbed against all other dimensions.
Vertical
Win rate across 8-vertical taxonomy aligned to TerritoryDefinitions and ICP Scorer.
Competitive scenario
Won outright / lost to named competitor / lost to no decision. Competitor breakdown feeds CalibrationSignalLog → AGT-403.
Rep
Individual win rate vs segment + vertical benchmark. Systematic underperformance feeds AGT-701 coaching input.
Deal size band
Win rate by ACV band (e.g. <$50K / $50–$150K / $150–$500K / $500K+). Identifies sweet spot and ceiling risk.
Sales cycle length
Actual cycle days vs segment benchmark. Deals won faster or slower than benchmark — directional signal for methodology and ICP fit.
Discount depth
Win/loss rate by discount tier. Identifies whether discounting is correlated with wins or signals desperation discounting on losing deals.
Stage lost
Where in the funnel did the deal die — Discovery / Technical Eval / Proposal / Negotiation / Close. Identifies methodology gaps by stage.
Persona coverage at close
Economic buyer engaged Y/N at deal close. Deals lost without economic buyer engagement are a distinct pattern — feeds ICP multi-threading signal.
Minimum sample sizes for pattern aggregation

Single-deal observations are noise. AGT-703 only surfaces a dimension pattern when minimum sample thresholds are met. Below threshold, data accumulates but no signal is generated.

Compared		 When both forecasts agree, confidence is high. When they diverge materially (>15% delta), the divergence pattern is analyzed — which model has been more accurate historically in this segment / period type? ForecastAccuracyLog (historical rows)

Bias detection model

Variance magnitude alone is insufficient. AGT-703 distinguishes between random variance and systematic bias.

Calibration output model

AGT-703 flags signals — humans decide. Three simultaneous outputs on every calibration signal. All three fire together; none replaces the others.

1
CalibrationSignalLog (write) — structured record of the signal: source module (win-loss or forecast), dimension, pattern detected, confidence level, recommended action type, owning agent or L0 table. Persists all signals with full traceability.
2
AGT-704 MBR surface — CalibrationSignalLog is pulled by AGT-704 at MBR synthesis. Calibration signals are a standing MBR section — not buried in appendix. RevOps and Finance see them in the monthly review cadence and decide whether to act.
3
Owning agent or L0 table notification — the agent or table that would need to change receives a direct flag. ICP signal → AGT-201 config flag. Forecast bias → AGT-402 or AGT-404 config flag. Competitive gap → AGT-403 KB flag. Quota methodology → AGT-101 flag. These are notification flags — the owning agent does not auto-update. Human approval required for any config change.
CalibrationSignalLog is the audit trail. Every signal ever generated is persisted with its status (open / actioned / dismissed) and who actioned or dismissed it. This creates traceability between observed pattern and the decision made — essential for evaluating whether calibrations improved outcomes.

Design guardrails
AGT-703 flags signals — humans decide. It never writes to ICP config, quota tables, competitive KB, or forecast model parameters. Every calibration signal requires a human to action it. CalibrationSignalLog status field enforces accountability: open signals with no action after 60 days are re-surfaced in the next MBR.
Low-confidence win-loss records are excluded from pattern aggregation. A pattern built on low-confidence data is worse than no pattern — it creates false calibration signals. Confidence gates are non-negotiable.
Rep commit accuracy is routed to AGT-701 as a coaching input — it is not a forecast credibility score that managers see independently. The goal is to improve rep forecasting behavior, not create a public accuracy leaderboard that incentivizes gaming.
AGT-703 does not re-open closed calibration signals that were dismissed. If a dismissed signal pattern resurfaces in a subsequent analysis cycle, a new signal is generated. This prevents stale signals from cluttering the log and ensures each cycle is evaluated on its own evidence.
