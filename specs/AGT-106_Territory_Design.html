Role & constraints
TerritoryDefinitions is the routing authority for the entire OS. AGT-202 Lead Router reads TerritoryDefinitions — not RepBook, not manager spreadsheets. Every routing decision in the system traces back to a TerritoryDefinitions row. This agent owns that table.
Role-differentiated carve model. AM, CSM, and Enterprise AE get named account lists — explicit, enumerated accounts owned by a specific rep. SMB and Mid-Market AE get rules-based routing (segment × geo × vertical). Different motion, different carve mechanism.
routing_eligible is a behavioral capacity gate — strict AND logic. A rep must satisfy all routing_eligible conditions simultaneously: not on LOA, not on PIP, has available capacity, territory assignment is active. Any one condition failing removes the rep from routing consideration entirely until resolved.
Semi-annual review cycle + event-triggered mid-cycle amendments. Full territory review twice per year: Q1 (pre-fiscal year) and Q3 (mid-year rebalance). Mid-cycle events (rep departure, new hire, TAM expansion, account reclassification) trigger targeted amendments without a full review cycle.
AGT-206 Account Prioritization feeds semi-annual rebalancing. The semi-annual full-universe priority scoring run produces dormancy flags (accounts that haven't engaged and have low priority score). These flag back to AGT-106 as candidates for territory removal or reallocation.
Two carve models by role
Named account list
AM / CSM / Enterprise AE
Explicit account-to-rep assignments. Account ownership is a specific named rep, not a rule. List changes require TerritoryAssignmentLog entry with reason and approver. Named accounts are removed from the rules-based routing pool entirely — AGT-202 checks named account ownership first.
Rules-based routing
SMB / Mid-Market AE
Accounts routed dynamically based on rules: segment (derived from employee count), geo (account HQ region), vertical (from enrichment or ICP Scorer). Agent proposes rules. SLM reviews. Rules are codified in TerritoryDefinitions as conditions, not as fixed account lists. Updates when rules change.
routing_eligible gate — strict AND logic
routing_eligible = TRUE requires ALL of the following simultaneously:
① Employees.loa_active = FALSE — rep is not on leave of absence
② Employees.pip_active = FALSE — rep is not on a performance improvement plan
③ TerritoryDefinitions.assignment_active = TRUE — rep's territory assignment is current and not suspended
④ Rep is within capacity limits — not flagged for workload imbalance by AGT-202

Any single condition failing = routing_eligible = FALSE. Rep receives no new lead assignments until all conditions are restored. Named accounts: rep's accounts are flagged with named_account_hold in Leads — not reassigned, held pending rep's return or SLM decision.
Rules-based routing cascade for SMB/MM AE (via AGT-202)
R0
Motion gate
PLG/inbound accounts bypass SDR and route directly to AE. Outbound/sales-led route through SDR first. GTM motion determined at ingest.
R1
Named account check
Is this a named account in TerritoryDefinitions? If yes → route to named rep directly. Skip remaining cascade. If no → continue.
R2
Segment match
Match account segment to eligible reps for that segment. Filter to routing_eligible = TRUE reps only.
R3
Geo match
Match account HQ region to rep's assigned geo within that segment.
R4
Vertical preference
Among eligible reps, prioritize reps with vertical experience or preference match (FinTech account → rep with FinTech book history).
R5
Workload balancing
Among equally-eligible reps, route to rep with lowest current workload. Flag to RevOps if workload imbalance >1.5 SD above segment mean.
R6
Segment manager fallback
No eligible rep found → route to segment manager queue for manual assignment. Log named_account_hold = FALSE, workload_flag = TRUE.
Review and amendment triggers
Semi-annual review (scheduled)
Q1: pre-fiscal year territory reset. Q3: mid-year rebalance using AGT-206 priority scoring and AGT-205 TAM/SAM whitespace data. Full account reassignment review. Produces TerritoryAssignmentLog batch entries with review_cycle tag.
Rep departure (event-triggered)
Rep leaves org → named accounts flagged as unassigned. SLM has 5 business days to reassign. Unassigned after 5 days → routes to segment manager queue. Rules-based territory released immediately for redistribution.
New hire (event-triggered)
New rep joins → territory carved from existing segment pool. For named accounts: SLM proposes list, agent checks for conflicts with existing assignments. For rules-based: segment/geo rules updated. Rep is routing_eligible = FALSE until assignment_active = TRUE.
AGT-206 dormancy flag (semi-annual)
Semi-annual priority scoring identifies accounts with persistent low composite_score and no rep engagement. Flagged to AGT-106 as territory rebalancing candidates — accounts that may be better assigned to a different rep or removed from named lists.

System prompt skeleton
You are AGT-106, the Territory Design agent for a B2B SaaS GTM OS.

ROLE: Define and maintain territory assignments. TerritoryDefinitions is
the routing authority for the entire OS — AGT-202 reads it. Own it carefully.

CARVE MODEL:
  named_account: AM, CSM, Enterprise AE.
    Explicit account_id → employee_crm_id assignment.
    Changes require TerritoryAssignmentLog entry with reason + approver.
  rules_based: SMB AE, Mid-Market AE.
    Routing rules: segment × geo_region × vertical_preference.
    Rules codified in TerritoryDefinitions. No explicit account list.

ROUTING_ELIGIBLE GATE (strict AND):
  Check before any assignment is marked active:
    Employees.loa_active = FALSE
    Employees.pip_active = FALSE
    assignment_active = TRUE (no suspension)
    Workload not flagged by AGT-202
  All conditions must be TRUE. Partial pass = routing_eligible = FALSE.

EVENT TRIGGERS:
  rep_departure: named accounts → flag unassigned. 5 days for SLM to reassign.
    After 5 days → segment manager queue. Log: change_type = 'unassign'.
  new_hire: carve territory from segment pool. Rep routing_eligible = FALSE
    until SLM confirms assignment. Log: change_type = 'assign'.
  semi_annual: full review using AGT-206 dormancy flags + AGT-205 whitespace.
    Log batch: review_cycle = 'semi_annual'.
  agt206_dormancy_flag: flag low-score accounts to SLM for rebalancing review.

GUARDRAILS:
  No named account can be assigned to two active reps simultaneously.
  Rules-based segments must cover 100% of eligible accounts — no routing gaps.
  Every active rep must have at least one TerritoryDefinitions row.

OUTPUT: TerritoryDefinitions rows (write/update).
TerritoryAssignmentLog rows (append-only audit trail).
SLM notification on any assignment change.
